# Домашнее задание 5 (Большое ДЗ 1)

Клиентская нода подбора паролей (параллельное программирование)

Цель работы

Разработать консольное приложение на C++, реализующее клиентскую вычислительную ноду для распределенной системы подбора ключей. Основной акцент — эффективное параллельное вычисление и корректная работа с сетевым взаимодействием.


## 1. Постановка задачи

Необходимо разработать программу, которая по заданной хэш-сумме вида:

`sha1(md5(key))`


выполняет поиск исходного значения key в заданном диапазоне.

Требования к ключу:

- Длина key: от 1 до 32 символов


Допустимые символы:

- [0-9a-zA-Z]


Диапазон поиска задаётся лексикографически, включая обе границы.


Пример диапазона:

- [A001; Z0002]


## 2. Формат программы


Программа должна быть реализована в виде консольного клиента.


Формат запуска:

```
./app <host> <port> <path_to_hashrate_file>
```

Где:

- `host` — адрес управляющего сервера
- `port` — порт сервера
- `path_to_hashrate_file` — путь к файлу, содержащему оценку производительности (количество вычислений sha1(md5(key)) в секунду)


## 3. Взаимодействие с сервером


Клиент работает по TCP/IP и должен:

- Подключаться к серверу
- Передавать серверу свою вычислительную производительность
- Принимать одно из состояний:
    - `TASK` — получить задачу на перебор диапазона
    - `WAIT` — ожидать указанное время
    - `DONE` — завершить работу


После выполнения задачи отправить результат:

- `FOUND <key>` — если ключ найден
- `NOT_FOUND` — если ключ отсутствует в диапазоне


*Важно*

Формат сетевого протокола не задаётся заранее.


Самостоятельно:

- разработать текстовый протокол взаимодействия клиента и сервера;
- определить формат сообщений;
- предусмотреть обработку ошибок;
- документировать принятые решения.


Минимальные требования: протокол должен позволять передавать производительность клиента, принимать задачу, команду ожидания, команду завершения и отправлять результат выполнения.


## 4. Требования к реализации


Разрешается использовать:

- C++
- стандартную библиотеку C++
- pthread
- POSIX sockets (C sockets)


Допускается комбинирование подходов, если оно технически обосновано.


Использование сторонних библиотек — только по согласованию.


## 5. Параллельность (ключевая часть работы)


Программа должна:

- Использовать многопоточность для ускорения перебора
- Рационально распределять диапазон между потоками


Обеспечивать корректную остановку всех потоков при:

- нахождении решения
- получении сигнала завершения (сигнал от ОС)


Необходимо обеспечить:

- отсутствие гонок данных
- корректную синхронизацию (mutex / atomic / condition variable и т.д.)
- масштабируемость по числу потоков


## 6. Обработка ошибок


Обязательно:

- Проверка корректности входных параметров
- Обработка ошибок подключения
- Обработка сетевых ошибок
- Обработка некорректных или неожиданных сообщений
- Проверка корректности диапазона


Программа не должна аварийно завершаться при штатных ошибках.


## 7. Завершение работы


Необходимо корректно обрабатывать:

- Ctrl+C (SIGINT)
- kill
- другие сигналы завершения


Требуется:


- Освобождение ресурсов
- Корректное завершение потоков
- Закрытие сетевых соединений
